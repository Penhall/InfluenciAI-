# Padrões de Desenvolvimento — Resumo Prático

Esta é uma visão executiva dos padrões; os detalhes completos estão em:

- `.rules/rules.md` (regras de desenvolvimento)
- `.rules/style_guide.md` (guia de estilo visual WPF)

Linguagem e documentação:

- Código, nomes, commits e PRs em inglês
- Documentação de negócio/técnica em português (BR)

Formatação e encoding:

- `.editorconfig` obrigatório; UTF-8; `crlf`; fim de linha final
- C# e XAML com BOM; indentação 4 spaces (C#/XAML), 2 (JSON/XML)

Arquitetura e padrões:

- Clean Architecture pragmática; MVVM no WPF
- DI para serviços; MediatR para orquestração no backend
- CQRS onde agrega clareza; Result Pattern para APIs

XAML e UI (WPF):

- Usar recursos e estilos globais do projeto
- Cores/Tipografia conforme `.rules/style_guide.md`
- Estados (hover/disabled/loading) e acessibilidade

Erros, Logs e Telemetria:

- Tratamento centralizado; logs estruturados com correlação
- Não expor detalhes sensíveis; mapear para mensagens de negócio

Testes e qualidade:

- xUnit/NUnit; testes de unidade e integração
- Cobertura mínima; testes de UI automatizados quando viável

Segurança:

- Princípio do menor privilégio; segredos no Key Vault
- Validação de entradas e sanitização; auditoria

Checklist de implementação (amostra):

- [ ] View com DataContext tipado e estilos globais
- [ ] Bindings explícitos e performáticos; listas virtualizadas
- [ ] Estados visuais (loading/error/disabled) cobertos
- [ ] Testes de unidade/integrados para fluxos críticos
